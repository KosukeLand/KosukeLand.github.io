1. はじめに

みなさま，はじめまして．当ブログにアクセスいただきありがとうございます．
筆者は某SIerに勤める6年目会社員で，主に基盤系SEとしてプライベートクラウド基盤の構築を中心に仕事をしてきました．

業務を通して様々な経験をし，それを自身の中で消化してきたつもりではありますが，
知識を正しく理解しているか，今一度振り返って確認してみたいと思い，当ブログを立ち上げました．

あくまで，自分自身の備忘録として利用するつもりではありますが，
私が遭遇したトラブル事例も記載しますので，私と同様のトラブルに遭遇した方のお役に立てればと考えています．

2. ブログを始めるにあたり

ブログを始めるにあたり，ツール選びから始めました．
私のブログに求める要件は以下の通り

  • 無料であること
  • サーバレスであること
  • ブログを通して何かしらの学びがあること
  • ジェネレータでindex.htmlを自動作成できること 　←フルスクラッチなんて無理w

以上の要件だとツールを絞りきれないのですが，今回はGithub.ioとHexoの組み合わせでブログを作成することにしました．
理由は，，，，

  • Github.ioを利用する理由 : Gitの勉強になるから(筆者はGit未経験である)
  • Hexoを利用する理由 : Node.jsで作られているから(筆者はjavascriptなら少しできる)

※Github.ioやHexoの説明は公式を確認してね
https://pages.github.com/
https://hexo.io/

7. その他

7.1. ディレクトリ構成

‘hexo init [Directory]’ をした際に作成されるファイルは以下の通り

  • /node_modules: 構成しているnpmライブラリの置き場所
  • /public: ここに公開用のデータ一式が書き出される
  • /scaffolds: 記事や固定ページを作るときのMarkdownテンプレート集
  • /source: 記事や固定ページを構成するMarkdown・画像データを置く場所
  • /themes: ブログのテーマが置かれる場所（デフォルトテーマ入り）
  • _config.yml: ブログのタイトルやURLなどの設定を書くファイル
  • .gitignore: Gitで無視するファイルやディレクトリを書くファイル
  • db.json: キャッシュが書き込まれるファイル
  • package-lock.json: 特定バージョンのライブラリ構成を共有する目的のファイル
  • package.json: Hexoを構成しているライブラリリスト

Reference: https://hexo.io/docs/
