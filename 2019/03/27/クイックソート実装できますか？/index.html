クイックソート実装できますか？

1. はじめに

先日，「クイックソートの最悪計算量はいくつ？」と聞かれ，
「O(NlogN)です！」と高らかに解答して恥ずかしい思いをした筆者です．こんにちは！

筆者の復習をかねて，今回はクイックソートを取り上げます．
それでは行きましょう．

2. クイックソート概要

クイックソートは，最良計算量O(NlogN)，平均計算量O(NlogN)，最悪計算量O(N^2)のアルゴリズムです．
計算量から分かる様に最悪のケースを引かなければ，とってもクイックにソート可能です．

クイックソート(昇順)は以下の手順で実施します．

  1. ソート対象の中で基準要素を決定する(ソート対象の中央値が望ましい)
  2. 基準要素よりも「小さい要素のグループ」を前方に，基準要素よりも「大きい要素のグループ」後方に移動する(分割する)
  3. 1,2を繰り返し，各グループの要素が1つになったら終了する

図で書くとこんな感じです(図1では，グループの先頭要素を基準要素としている)

2.1 クイックソート最良計算量

クイックソートで最良計算量となるのは，常にN/2:N/2で分割されるケースで
その場合，入れ替え捜査のために配列をNlogN回捜査する必要があるため，実行時間はO(NlogN)となります．
※ NlogN = N + (N/2 + N/2) + (N/4 + N/4 + N/4 + N/4) + … +(2 + 2 + … + 2)

2.2 クイックソート最悪計算量

クイックソートで最悪計算量となるのは，常に1:N-1で分割されるケースで
その場合，入れ替え操作のために配列内をN(N-2)/2回捜査する必要があるため，実行時間はO(N^2)となります．
※ N(N+2)/2 = N + N-1 + N-2 + … + 2
